# Starter Packs

Starter packs group the repository into simpler entry points so teams can adopt it without browsing all 93 skills at once.

---

## 1. PM Core

**Best for:** most product managers

**Plugins**
- `pm-product-discovery`
- `pm-product-strategy`
- `pm-execution`

**Best starting commands**
- `/discover`
- `/strategy`
- `/write-prd`
- `/write-stories`
- `/sprint`

**Use this when**
- you want a default PM operating system
- your team is not focused on AI features yet
- you want consistent artifacts across discovery, planning, and delivery

---

## 2. AI PM

**Best for:** PMs shipping AI features

**Plugins**
- `pm-ai-product-management`
- `pm-data-analytics`
- `pm-product-strategy`
- `pm-execution`

**Best starting commands**
- `/ai-feature-launch`
- `/ai-model-eval`
- `/responsible-ai-review`
- `/ai-metrics`
- `/write-prd`

**Use this when**
- you need model selection, fallback logic, and responsible AI coverage
- you want clearer AI metrics and incident planning
- you need a safer feature launch workflow

---

## 3. PM Builder

**Best for:** PMs prototyping with AI coding tools

**Plugins**
- `pm-vibe-coding`
- `pm-execution`
- `pm-ai-product-management`

**Best starting commands**
- `/plan-prototype`
- `/vibe-spec`
- `/tech-decision`
- `/review-code`
- `/deploy-check`

**Use this when**
- you are building MVPs with Cursor, Claude Code, Copilot, or Windsurf
- you need product-safe guidance before shipping
- you want structure before asking an AI coding agent to build

---

## 4. PM Learning

**Best for:** new PMs and team onboarding

**Plugins**
- `pm-guided-learning`
- `pm-execution`
- `pm-product-discovery`

**Best starting commands**
- `/course`
- `/learn-foundations`
- `/learn-execution`
- `/learn-capstone`

**Use this when**
- you are onboarding junior PMs
- you want a reusable PM training path
- you want learning plus practical artifact creation

---

## Suggested adoption order

1. **PM Core**
2. **AI PM** or **PM Builder** depending on your work
3. **PM Learning** for team enablement
