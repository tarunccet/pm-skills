# pm-data-analytics

Data analytics skills for PMs: SQL query generation, cohort analysis, A/B test analysis, product metrics framework, funnel analysis, event tracking planning, metric definition, and North Star metric definition.

## Skills (7)

- **ab-test-analysis** — Analyze A/B test results with statistical significance, sample size validation, confidence intervals, and actionable recommendations.
- **cohort-analysis** — Perform cohort analysis on user engagement data.
- **product-metrics** — Define a complete product metrics framework: North Star metric selection (with business game classification and 7-criteria validation), input metrics, health metrics, counter-metrics, dashboard design, alert thresholds, and — for AI products — a four-layer AI metrics stack (model quality, operational, product-level, business impact).
- **sql-queries** — Generate SQL queries from natural language descriptions.
- **funnel-analysis** — Analyze conversion funnels: identify drop-off points, calculate stage-by-stage conversion rates, generate leakage hypotheses, and recommend improvement experiments.
- **event-tracking-plan** — Design an analytics instrumentation plan: define key events, properties, naming conventions, and produce a tracking spec document for engineers.
- **metric-definition** — Define and distinguish operational, vanity, and actionable metrics. Write full metric specs with name, definition, data source, formula, owner, and review cadence.

## Commands (6)

- `/pm-data-analytics:analyze-cohorts` — Perform cohort analysis on user data — retention curves, feature adoption, and engagement trends.
- `/pm-data-analytics:analyze-test` — Analyze A/B test results — statistical significance, sample size validation, and ship/extend/stop recommendations.
- `/pm-data-analytics:write-query` — Generate SQL queries from natural language — supports BigQuery, PostgreSQL, MySQL, and more.
- `/pm-data-analytics:north-star` — Define your North Star Metric and supporting input metrics — classify the business game and validate against best practices.
- `/pm-data-analytics:design-funnel` — Analyze a conversion funnel and get drop-off insights and improvement recommendations.
- `/pm-data-analytics:plan-tracking` — Design an analytics event tracking plan with event taxonomy, property schemas, and naming conventions.
