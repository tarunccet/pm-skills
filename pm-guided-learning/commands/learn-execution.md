# /learn-execution — Product Execution & Delivery Module

Launches the interactive learning module on product execution — the craft of translating strategy and discovery into shipped, working products.

## Invocation

```
/learn-execution                    # start the module
/learn-execution beginner           # start with PRD template scaffolding
/learn-execution advanced           # start in critique mode — write first, then get feedback
```

## Workflow

Load and execute the `learn-execution` skill. The module will:
1. Brief you on the Notely scenario (note-taking app with a retention goal)
2. Walk you through writing a 6-section PRD for the Daily Digest feature
3. Run Quiz Checkpoint 1 (PRD best practices)
4. Convert a feature-based roadmap into an outcome-based roadmap
5. Run Quiz Checkpoint 2 (roadmap design)
6. Write 2 OKRs for the product team
7. Run a mini sprint planning simulation with engineer questions
8. Run Quiz Checkpoint 3 (OKR quality and sprint planning)
9. Provide a final execution debrief with scores and next-step recommendations

## Edge Cases

- If the learner has never written a PRD: provide the 6-section template upfront and walk through each section with a brief explanation before asking them to fill it in.
- If the learner asks to use a different PRD format (e.g., Amazon one-pager): acknowledge the format, note that this module teaches the 6-section structure as a baseline, and explain that both formats share the same goals — just adapt the feedback accordingly.
- If the learner says their company doesn't use OKRs: explain that OKRs are a proxy for any outcome-focused goal-setting — the principles apply even if the label is different.
- If the learner's PRD is already excellent: skip the section-by-section guidance and jump directly to comprehensive critique mode.
